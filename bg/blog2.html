What is Github?

Github is an internet hosting service for software development and others.It's an service where user can summon their different projects,contents etc.in files called as Repository.The site provide social-networking features such as followers and a graph to display how user work on their Repository.

Create a repository:

Go to Github make your first repository.You can make your repository private or public to all to showcase your projects. Repository is simply modified name of a folder that contains your projects files,image etc.

Clone A repository:

Cloning can be simply be defined as copying.You can clone a repository from Github.comor by passing the above git command in terminal to your local machine.

Cloning a repository copies down all the repository data including all files,folder of the project.By this you can also contribute to someone repository such as by fixing an issue in other repository etc.

Pushing in Github:

An owner can make changes to it repository files by adding ,commiting and then pushing the changes to original files.WE can do all these operations by the above git commands,

Commit in github is a certified text which define about the changes the user done to its files . Pushing means the user finally want to merge its change to the original files.

Branch in Github:

A branch is a version of the repository which allow user to develop,fix an issue or bug which is in a repository.A project contains a multiple branch.When you want to alter or fix an bug You create an new branch to summarize your changes. Afterwards the changes you made is merged into a default branch named as Master branch. A repository contain only one single master branch.

Pulling request:

Pull requests are features that let you tell other about changes you' made to repository on Github. By pull request you can fix an issue on other repository.

Working of Pull Request-

  • A developer creates the solution in branch in their local repo.
  • After that,the branch is pushed to the public Github repository.
  • The developer than files a pull request via Github
  • The one which is owner of the repo review the changes and then alter it acccordingly
